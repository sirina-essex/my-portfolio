Research Proposal Review

Unit 3

My capstone project, titled “Identity and Access Governance in Microsoft Azure: Experimenting IAM Strategies through Entra ID in a Vulnerable Cloud Environment Based on Zero Trust Principles”, aims to assess the effectiveness, granularity, and governance potential of native IAM mechanisms in Azure. The project uses the deliberately vulnerable open-source application Broken Crystal as a testbed to simulate realistic yet insecure access conditions.

1. Type of Research

This research follows a threefold approach:

  • Exploratory, as it investigates the concrete implementation of advanced IAM strategies (RBAC, PIM, CA, MFA, Access Reviews) in a complex and under-documented context.
  • Experimental, since it involves configuring and manipulating security policies in a controlled environment.
  • Evaluative, because it seeks to assess the actual impact of these mechanisms on access security, governance, and traceability, in reference to Zero Trust principles (NIST SP 800-207).

2. Research Design

The research adopts a mixed-methods design, combining:

  • A case study based on Broken Crystal, integrated into an Azure tenant;
  • A controlled experiment, where each IAM mechanism is deployed in isolation or in combination;
  • Simulated scenarios, driven by realistic personas (internal collaborator, external developer, malicious actor …);
  • A comparative approach, to evaluate the relative effectiveness of different IAM configurations (e.g., RBAC alone vs RBAC + PIM).

Each test follows a structured scenario, with clear deployment, execution, and observation phases ensuring full traceability.

3. Data Collection Methods and Data Types

The data collection strategy includes:

  • Direct observation, to document visible outcomes (access denial, privilege escalation, triggered alerts);
  • System logs (Azure Monitor, Log Analytics, Defender for Cloud), capturing quantifiable events;
  • Automated captures of dashboards and reports;
  • Documentary analysis, drawing on Microsoft documentation and regulatory frameworks (NIST ZTA, Entra ID official guidance).

Data types include:

  • Primary data, generated through testing;
  • Secondary data, from technical documentation and academic sources;
  • Quantitative data (event frequency, timestamps, access attempts …);
  • Qualitative data, (policy configurations, log content, unexpected behaviors).

4. The analysis combines descriptive, comparative, and cross-sectional methods, with:

  • Triangulation between (1) the deployed IAM configuration, (2) the simulated action, (3) the system's actual response;
  • An evaluation framework inspired by the Zero Trust Architecture model , assessing least privilege, continuous verification, auditability, and denial of implicit trust.

The objective is not to validate a full Zero Trust Architecture but to determine to what extent Azure’s IAM mechanisms align with its core principles in a controlled, vulnerable test environment.

5. Required and Targeted Skills

This project requires to mobilize and develop:

  • Technical expertise in cloud IAM (Entra ID, PIM, CA, MFA);
  • Operational knowledge of Azure Monitor, Log Analytics, Defender for Cloud
  • The ability to structure reproducible scenarios and analyze dynamic IAM policies,
  • A deep understanding of regulatory frameworks, (NIST SP 800-207, ISO/IEC 27001).
  • A rigorous, evidence-based approach to experimental documentation and critical analysis.
